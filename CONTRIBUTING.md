# Contributing to Murmur API

Thank you for your interest in contributing to the Murmur API project!

## Development Workflow

1. **Fork and Clone**: Fork the repository and clone it to your local machine
2. **Branch**: Create a feature branch from `develop`
3. **Develop**: Make your changes following our code standards
4. **Test**: Ensure all tests pass
5. **Commit**: Write clear, descriptive commit messages
6. **Pull Request**: Submit a PR to the `develop` branch

## Code Standards

- Follow the existing code style
- Write meaningful variable and function names
- Add comments for complex logic
- Include unit tests for new features
- Update documentation as needed

## Testing

```bash
# Run unit tests
npm run test

# Run e2e tests
npm run test:e2e

# Run test coverage
npm run test:cov
```

## Commit Message Guidelines

- Use present tense ("Add feature" not "Added feature")
- Use imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit the first line to 72 characters or less
- Reference issues and pull requests liberally

## Questions?

Feel free to open an issue if you have any questions about contributing!
